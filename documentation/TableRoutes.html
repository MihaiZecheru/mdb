Requests for the /tables/ endpoint

General notes

The /tables/ endpoint is part of the ‘backend’ for this API, as this table holds what is essentially ‘metadata’ for the application.

There are two database tables for keeping track of a user’s custom tables. One of the tables keeps track of which tables a user owns and how many there are, while another keeps track of the metadata for each table. Operations can be performed on the tables table through the API calls in this collection.

The tables themselves are what make up the custom API. A user sends a post request to the /tables/ endpoint to create an API, and from there, the user can interact with their new API by sending requests to the /api/ endpoint. Each custom API is an SQL table that can be interacted with via HTTP requests.

Tables are private, meaning only the user who owns them can perform operations on them. To confirm you are the user making a request to this endpoint, provide your API key / access token in the request headers.

This table can be accessed by regular users, but it is mostly used behind the scenes when users access their custom APIs - this is what is meant by the ‘backend’.

Schema

Field Type Description
owner_id int the user_id of the table owner
table_id string (63) the table’s name
environment_name string (25) the name of the environment the table belongs to
tablename varchar (31) the table’s name
description varchar (500) a summary / description of the table
fields array[field]

field {
name,
type,
setNotNull,
default,
autodate
}		 contains each field that makes up the custom API table.

Note: get table count is not available from this collection. This request is a part of the /users/ collection.

API detail

Create New Table

Create a new table under the environment with the given env_name belonging to the user with the given user_id.

The table information–the name, description, and fields–must be passed into the request body.

Returns the new table or an error message.

  • Curl
  • HTTP
curl -X POST -H "Authorization: {{auth}}" -d '{
    "table_name": "users",
    "table_description": "table for testing the api",
    "fields": {
        "username": "string_30",
        "age": "integer",
        "float_number": "float",
        "is_cool": "boolean",
        "arr": "array",
        "json_field": "json",
        "emoji_field": "emoji",
        "made": {
            "type": "datetime",
            "auto_date": true
        }
    }
}' "https://mdb.mzecheru.com/tables/{{user_id}}/{{env_name}}"

Update Table

Update the table with the given table_name under the environment with the given env_name belonging to the user with the given user_id.

The table information–the name, description, and fields–must be passed into the request body.

  • Use rename_fields to change the name of existing fields.
  • Use add_fields to add new fields to the table.
  • Use remove_fields to delete existing fields.

Returns the updated table or an error message.

  • Curl
  • HTTP
curl -X PATCH -H "Authorization: {{auth}}" -d '{
    "rename_fields": {
        "password": "open_sesame!"
    }
}' "https://mdb.mzecheru.com/tables/{{user_id}}/{{env_name}}/{{table_name}}"
PATCH /tables/%7B%7Buser_id%7D%7D/%7B%7Benv_name%7D%7D/%7B%7Btable_name%7D%7D HTTP/1.1
Host: mdb.mzecheru.com
Authorization: {{auth}}

{
    "rename_fields": {
        "password": "open_sesame!"
    }
}
