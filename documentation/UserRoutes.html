Requests for the /users/ endpoint

General notes

The /users/ endpoint is part of the ‘backend’ for this API, as this table holds what is essentially ‘metadata’ for the application.

The users table holds information on the users who have made an account with MDB. Operations can be performed on the table through the API calls in this collection.

Users own environments, and environments each own a series of tables. Therefore, the user’s linked environments, linked tables, environment count, and table count are accessed from this endpoint, not from the /environments/ or /tables/ endpoints.

In order to create a new user, an admin API key / access token must be provided. Users can access their own entry in the database by confirming they are the user they wish to perform an operation on by providing their API key / access token in the headers of their request to this endpoint. Users may not perform operations on another user’s data / entry.

This table can be accessed by regular users, but it is mostly used behind the scenes when users access their custom APIs - this is what is meant by the ‘backend’.

Schema

Field Type Description
id int the user’s identifier
username string (50) username
password string (255) password
email string (320) email
auth string (72) the user’s access token / authentication

API detail

Create New User

Create a new MDB user so that they can begin to create environments and tables to interact with.

The username, password, email fields must be passed into the body of the request. The id and auth fields will be automatically filled for the user.

This request will return the new user or an error message.

  • Curl
  • HTTP
curl -X POST -H "Authorization: {{admin_auth}}" -d '{
    "username": "chrissy",
    "password": "mihai28!",
    "email": "chrissy@gmail.com"
}' "http://localhost:3000/users/"
POST /users/ HTTP/1.1
Host: localhost:3000
Authorization: {{admin_auth}}

{
    "username": "chrissy",
    "password": "mihai28!",
    "email": "chrissy@gmail.com"
}
  • Response
Status200 OK
Content-Typeapplication/json
{
    "id": 741,
    "username": "example_user",
    "password": "abc123",
    "email": "email@example.com",
    "auth": "d84b9543-ea39-4134-a7e7-78992400a15c"
}

Update User Fields

Update the MDB user with the given user_id.

The username, password, and email fields must be passed into the body of the request. Note: the auth and id fields are permanent and cannot be changed.

The request will return the updated user or an error message.

  • Curl
  • HTTP
curl -X PATCH -H "Authorization: {{auth}}" -d '{
    "username": "chrissy",
    "password": "mihai28!",
    "email": "chrissy@hotmail.com"
}' "http://localhost:3000/users/{{user_id}}"
PATCH /users/%7B%7Buser_id%7D%7D HTTP/1.1
Host: localhost:3000
Authorization: {{auth}}

{
    "username": "chrissy",
    "password": "mihai28!",
    "email": "chrissy@hotmail.com"
}
  • Response
Status200 OK
Content-Typeapplication/json
{
    "id": 741,
    "username": "example_user_new_name",
    "password": "abc123",
    "email": "email@example.com",
    "auth": "d84b9543-ea39-4134-a7e7-78992400a15c"
}

Get User Linked Tables

Get the linked table_ids the user with the given user_id has.

Linked tables are the tables a user owns, regardless of the environment that they are in.

Returns an array of the linked table_ids or an error message.

  • Curl
  • HTTP
curl -X GET -H "Authorization: {{auth}}" "http://localhost:3000/users/{{user_id}}/tables"
GET /users/%7B%7Buser_id%7D%7D/tables HTTP/1.1
Host: localhost:3000
Authorization: {{auth}}
  • Response
Status200 OK
Content-Typeapplication/json
[
    "_example_env_example_table",
    "_example_env_example_table1"
]
