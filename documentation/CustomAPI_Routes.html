Requests for your custom API endpoints

General notes

To successfully interact with the /api/ endpoint of the MDB application, a user must first create an environment, and then a table inside said environment. This table is what is being queried when requests are made to this endpoint.

Tables are made by setting up custom fields with properties such as setNotNull ,default ,auto_date etc. which can be used to customize each individual field in a table. Once the table is made, the user can interact with their table by following the standard REST API protocol:

  • new entries to the table can be created via POST requests
  • entries can be retrieved via GET requests
  • entries can be updated via PATCH requests
  • entries can be delete via DELETET requests

An example of a use case for this API is a userdata table, where each entry contains user data. Custom fields can be set up by initializing fields for a username and password, and each entry can be an individual user for your application.

Tables are private, meaning only the user who owns them can perform operations on them. To confirm you are the user making a request to this endpoint, provide your API key / access token in the request headers.

Custom Table API Example Requests

The access token is left out of the requests in this example.

The URL template is /api/{{user_id}}/{{env_name}}/{{table_name}}/

Method Request URL Request Body Response
POST /api/17/cool_api/userdata/ { username: DokZayas, password: PcRHgkY6Pv } UserdataEntry { _id: 7, username: DokZayas, password: PcRHgkY6Pv }
GET /api/17/cool_api/userdata/3 UserdataEntry { _id: 3, username: JohnDoe_2408, password: ar8cp1EUx21R4 }
PATCH /api/17/cool_api/userdata/5 { username: 99percentAvocado } UserdataEntry { _id: 5, username: 99percentAvocado, password: 1BPLaPJr }
DELETE /api/17/cool_api/userdata/2 { "message": "Entry with id '2' has been successfully deleted" }

Note: the provided examples will not actually work, and will return an authorization error as they are intended to be only visual examples.

Update Entry

Update the entry with the given entry_id in your custom API / database. The request URL contains the path to your table; /user_id/env_name/table_name/, ending with the entry number you wish to delete.

EndFragment

This post request cannot be properly defined other than in the URL, as the request body will contain different information for different tables. If, for example, a table contained two fields such as in the example above–username and password–the request body would contain values for a username and password.

Returns the updated entry or an error message.

  • Curl
  • HTTP
curl -X PUT -H "Authorization: {{auth}}" -d '{
    "username": "johnsmith"
}' "http://localhost:3000/api/{{user_id}}/{{env_name}}/{{table_name}}/{{entry_id}}"
PUT /api/%7B%7Buser_id%7D%7D/%7B%7Benv_name%7D%7D/%7B%7Btable_name%7D%7D/%7B%7Bentry_id%7D%7D HTTP/1.1
Host: localhost:3000
Authorization: {{auth}}

{
    "username": "johnsmith"
}

Create New Entry

Create a new entry to your custom API / database. The request URL contains the path to your table; /user_id/env_name/table_name/

This post request cannot be properly defined other than in the URL, as the request body will contain different information for different tables. If, for example, a table contained two fields such as in the example above–username and password–the request body would contain values for a username and password.

Returns the new entry or an error message.

  • Curl
  • HTTP
curl -X POST -H "Authorization: {{auth}}" -d '{
    "entry_fields": "are specific to each table / custom API",
    "but_follow": "this format of key-value pairs"
}' "http://localhost:3000/api/{{user_id}}/{{env_name}}/{{table_name}}"
